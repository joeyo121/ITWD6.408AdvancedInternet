XML RULES

6 important rules that must be adhered to in order to create a valid XML file:

Rule 1: All XML elements must have a closing tag
In XML, it is illegal to omit the closing tag. All elements must have a closing tag.
Example:
✔ <message>This is correct</message> or <br />
❌ <message>This is NOT correct<contact>

Rule 2: XML tags are case sensitive
Opening and closing tags must be written with the same case.
Example:
✔ <message>This is correct</message>
❌ <message>This is NOT correct</MeSsAgE>

Rule 3: All XML elements must be properly nested
In XML, all elements must be properly nested within each other
Example:
✔ <message><div>This is correct</div></message>
❌ <message><div>This is NOT correct</MeSsAgE><div>

Rule 4: All XML documents must have a root element
XML documents must contain one root element that is the parent of all other elements

Example:
✔ <root>
<child>
</child>
</root>

Rule 5: Attribute values must always be quoted
XML elements can have attributes in name/value pairs just like in HTML.
Example:
✔ <note date="22/2/22"> <message>This is correct</message> </note>
❌ <note> <message>This is NOT correct</message> </note>

Rule 6: Attribute values must be put in quotation marks
Specify a property for an element
Example:
✔ <message category="work">This is correct</message>
❌ <message category=work>This is NOT correct</message>
